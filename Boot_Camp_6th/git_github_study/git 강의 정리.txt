[2. git 개요]
- 버전 관리 시스템
- 리누스 토르발스가 만듬 ( 리눅스 창시자 )
- 특징 : 
	로컬 저장소 및 원격 저장소 생성 : git init
	로컬 저장소에 버전관리할 파일을 생성 및 추가 : git add ( 아직까지는 버전 생성 안됨 )
	버전관리할 파일의 버전 생성 : git commit + 커밋 메세지가 반드시 설정(ex. "update 코드 추가")
	버전 히스토리 확인 : git log
	버전간 비교 : git diff 
	특정버전으로 되돌아가기 : git reset, git revert
	새로운 브랜치 생성 ( 이전 브랜치에 영향을 및지지 않음 ) : git branch 브랜치명
	브랜치 병합 : git merge	
	로컬 저장소에 브랜치를 원격 저장소에 보내기 가능 : git push

7. 변경사항 비교
				git diff
	----------------------------------------------------------------------------

작업 디렉터리 -------------------------> 스테이지 -------------------------> 로컬 저장소
				   <------------------------------------>	해시값3 (HEAD)
					git diff --staged		해시값2
								해시값1
	        <------------------------------------------------------------------->
			git diff HEAD ( git diff + git diff --staged )
				

- git diff: 작업 디렉터리와 저장소 비교
- git diff --staged : 스테이지와 저장소비교
- git diff HEAD : git diff + git diff --staged 모두 포함
- git diff 해시값1 해시값2 : 해시값1, 2를 비교한다

	실습:
	$ mkdir git_diff
	$ cd git_diff
	$ git init	# .git 폴더 생성
	$ touch hello.txt
	$ echo "hello" >> hello.txt
	$ git add hello.txt
	$ git commit -m "first commit"
	
	# 작업 디렉터리 수정
	$ echo "hello2" >> hello.txt
	diff --git a/hello.txt b/hello.txt
	index ce01362..97531f3 100644
	--- a/hello.txt
	+++ b/hello.txt
	@@ -1 +1,2 @@
	 hello
	+hello2

	# 스테이지와 저장소 비교
	$ git add hello.txt
	$ git diff --staged
	// 이때 git diff 만 하면 작업 디렉토리와 스테이지 사이에 변경점이 없으므로 아무것도 뜨지 않는다.
	$ git diff --staged
	diff --git a/hello.txt b/hello.txt
	index ce01362..97531f3 100644
	--- a/hello.txt
	+++ b/hello.txt
	@@ -1 +1,2 @@
	 hello
	+hello2

	# 작업 디렉터리( hello3 ) + 스테이지( hello2 ) + 저장소 비교( hello )
	$ echo hello3 >> hello.txt	// 디렉터로와 스테이지의 변경점 추가
	$ git diff HEAD
	diff --git a/hello.txt b/hello.txt
	index ce01362..3cea797 100644
	--- a/hello.txt
	+++ b/hello.txt
	@@ -1 +1,3 @@
	 hello
	+hello2	// 스테이지 <-----> 저장소
	+hello3	// 작업 디렉토리 <-----> 스테이지
	
	# 해시값 비교
	$ git commit -m "hello2 commit"
	$ git add hello.txt
	$ git commit -m "hello3 commit"

	$ git log --oneline
	fee14ba (HEAD -> main) hello3 commit
	4eb113e hello2.txt
	53225b3 first commit


	$ git diff fee14ba 53225b3	// 첫번째 커밋과 가장 최신 커밋을 비교
	diff --git a/hello.txt b/hello.txt
	index 3cea797..ce01362 100644
	--- a/hello.txt
	+++ b/hello.txt
	@@ -1,3 +1 @@
	 hello
	-hello2	
	-hello3	// hello2, hello3 가 누락되어 있다.

9.  작업 되돌리기 

						
						
	작업 디렉터리 -------------------------> 스테이지 -------------------------> 로컬 저장소
	hello.txt 수정			         <------------------------->
						git reset --soft HEAD^ | 해시값 ( git commit 되돌리기 )
							   // HEAD^ : 가장최근 해시값
			  <------------------------------------------------------->
				git reset --mixed HEAD^ | 해시값 	( git add 되돌리기 )
	         <------------------------------------------------------------------------>
				git reset --hard HEAD^ | 해시값 	( 작업디렉터리 수정 전까지 되돌리기 )

	# 커밋 이전 상태 되돌리기

	git restore hello.txt	 	<== 작업 디렉토리의 작업 되돌리기
	git restore --staged hello.txt 	<== git add 되돌리기
	

	# 커밋 이후 상태 되돌리기 ( 이전 커밋 히스토리 정보가 삭제됨 )
	git reset --soft HEAD^ | 해시값 	( git commit 되돌리기 )
	git reset --mixed HEAD^ | 해시값 	( git add 되돌리기 )
	git reset --hard HEAD^ | 해시값 	( 작업디렉터리 수정 전까지 되돌리기 )

	실습:
	$ mkdir git_reset
	$ cd git_reset
	$ git init
	$ touch hello.txt
	$ echo "hello" >> hello.txt
	$ git add hello.txt
	$ git commit -m "hello commit"

	$ echo "hello2" >> hello.txt
	$ git commit -am "hello2 commit"

	$ echo "hello3" >> hello.txt
	$ git commit -am "hello3 commit"

	$ git log --oneline
	1d054c8 (HEAD -> main) hello3 commit

1. 기본 브랜치 
	- main 브랜치가 자동으로 제공 ( git init 할 때 )
	- 과거에는 master 브랜치가 기본이었음. 현재는 사용 안함
	- 각 브랜치는 독립적으고 관리된다!!!!!!!!!!!!

2. 기본 명령어 
	$ git branch : 브랜치 목록 보기
	* main	// * 은 현재 작업 중인 브랜치를 의미한다
	$ git branch 브랜치명 : 새로운 브랜치 생성
	$ git branch -M 브랜치명 : 브랜치 이름 변경, 변경할 브랜치에서 작업
	$ git branch -D 브랜치명 : 브랜치 강제 삭제, 다른 브랜치에서 작업
	$ git switch 브랜치명 : 브랜치 변경
	$ git checkout 브랜치며: 브랜치 변경 (OLD)

	$ git switch -C 브랜치명 : 브랜치를 생성하면서 해당 브랜치로 변경됨
	$ git log main..hotfix : main 기준으로 hotfix 차이점 출력
	$ git merge 브랜치명 : 현재 브랜치 기준으로 입력된 브랜치를 병합한다.



	실습:
 	$ mkdir git_branch
	$ cd git_branch
	$ git init
	$ touch hello.txt
	$ echo hello >> hello.txt
	$ git add hello.txt
	$ git commit -m "hello commit"	// 최초의 커밋이 있어야 main 브랜치가 생성된다

	$ git branch hotfix	# hotfix 이름의 새로운 브랜치 생성
	$ git branch
	  hotfix	// 새롭게 생성된 브랜치
	* main
	
	# main 브랜치에서 작업
	$ git branch -M master
	
	# main 브랜치에서 작업
	$ git branch -D hotfix

	# main 브랜치에서 hotfix로 변경
	$ git switch hotfix
	$ git checkout hotfix

	# feature 브랜치 만들면서 변경
	$ git switch -C feature  

	# hotfix에 새로운 작업 추가
	$ git switch hotfix
	$ echo "hello2" >> hello.txt
	$ git commit -am "hello2 commit" <== main과 hotfix가 다름.

	$ git log --oneline
	4ce7a24 (HEAD -> hotfix) hello2 commit
	e638c78 (main, feature) hello commit

	# main 기준으로 hotfix 차이점 출력
	$ git log main..hotfix
	

	# main에서 작업. main에 브랜치변경 작업을 병합
	$ git merge hotfix
	$ git branch -D hotfix <==  병합후에는 hotfix 브랜치 삭제, git reflog로 복구 가능 but 복잡하다

3. 병합 종류

1) Fast-forward ( 빨리감기 병합 ) 
 main브랜치에서 분기한 후 (hotfix), hotfix에서 작업을 하고, main 브랜치에서는 추가작업이 병합했을 때
 fast-forward 방식으로 병합	// main은 아무작업을 안했을때, 메인의 HEAD를 병합하려는 브랜치의 최신 커밋으로 옮기는방식

	실습 : 
	$ mkdir git_merge
	$ cd git_merge
	$ git init
	
	# main 작업
	$ touch hello.txt
	$ echo "hello" >> hello.txt
	$ git add .
 	$ git commit -m "hello commit"	<-- HEAD
	$ git branch hotfix
	$ git switch hotfix

	# hotfix 작업
	$ echo "hello2" >> hello.txt	// main과 싱크가 안맞기 시작한다
	$ git commit -am "hello2 commit"	<-- HEAD
	$ git branch main

	# main 작업
	$ git merge hotfix	# Fast-forward 메시지 확인	// main의 HEAD가 병합하려는 hotfix의 헤드로 옮겨갔다.

	Updating 3822478..83ac846
	Fast-forward	// fast forward 발생
	 hello.txt | 1 +
	 1 file changed, 1 insertion(+)
