<h2 id="til-git">[TIL] Git</h2>
<p>오늘은 git의 작동 흐름과 그 과정에서 사용되는 명령어들을 배울 수 있었다. 오늘 배운 내용을 각 각 상태간의 상호작용 위주로 정리해 보았다</p>
<hr />
<h3 id="🟢-part-1-작업-디렉터리-→-스테이지">🟢 Part 1. 작업 디렉터리 → 스테이지</h3>
<p>파일을 수정하고, 다음 버전에 포함할 파일들을 Stage에 전달하는 과정입니다.</p>
<ul>
<li><strong><code>git add [파일명]</code></strong> : 파일을 스테이지에 올립니다. 이때부터 Git이 파일을 관리하는 'Tracked' 상태가 됩니다.</li>
<li><strong><code>git status</code></strong> : 현재 파일들이 어느 단계(Untracked, Modified, Staged)에 있는지 진단합니다.</li>
<li><strong><code>git restore [파일명]</code></strong> :  작업 디렉터리의 수정을 취소하고 마지막 커밋 상태로 되돌립니다.</li>
<li><strong><code>git reset [파일명]</code></strong> :  스테이지에 올라간 파일을 다시 작업 디렉터리로 내립니다.</li>
</ul>
<blockquote>
<p><strong>🛠️ 제외 관리 ( .gitignore )</strong>
<code>git add .</code>을 할 때 보안 파일이나 불필요한 파일이 포함되지 않도록 미리 설정하는 것. 아래 링크를 통해 현재 사용중인 OS, IDE, 언어에 맞춰 .gitignore 파일을 작성할 수 있다.</p>
<ul>
<li><strong>링크 :</strong> <a href="https://www.toptal.com/developers/gitignore">gitignore.io (Toptal)</a></li>
</ul>
</blockquote>
<hr />
<h3 id="🟡-part-2-스테이지-→-로컬-저장소">🟡 Part 2. 스테이지 → 로컬 저장소</h3>
<p>스테이지에 모인 변경 사항을 로컬 저장소에 기록하는 과정입니다.</p>
<ul>
<li><strong><code>git commit -m &quot;메시지&quot;</code></strong> : 스테이지의 내용을 로컬 저장소에 저장합니다.</li>
<li><strong><code>git commit --amend</code></strong> : 방금 만든 커밋에 오타가 있거나 파일을 빠뜨렸을 때, 기존 커밋을 덮어씁니다.</li>
<li><strong><code>git reset --soft [HEAD^ | 해시]</code></strong> :  커밋만 취소하고, 파일 내용은 <strong>스테이지에 그대로 남겨둡니다.</strong></li>
<li><strong><code>git reset --mixed [HEAD^ | 해시]</code></strong> :  커밋과 스테이징을 모두 취소하지만, <strong>작업 디렉터리의 파일 내용은 유지합니다.</strong></li>
</ul>
<blockquote>
<p><strong>💡 HEAD^ 의 의미</strong></p>
<ul>
<li><strong><code>HEAD</code></strong> : 현재 브랜치의 <strong>가장 최신 커밋</strong>을 가리키는 포인터입니다.</li>
<li><strong><code>^</code></strong> : '하나 전'을 의미하며 <code>HEAD^</code>는 최신 커밋의 바로 이전 버전을 뜻한다.</li>
</ul>
</blockquote>
<hr />
<h3 id="🔵-part-3-로컬-저장소-내부">🔵 Part 3. 로컬 저장소 내부</h3>
<p>커밋을 삭제 하거나 브랜치를 통해 버전을 관리하는 하는 단계</p>
<ul>
<li><strong><code>git reset --hard [HEAD^ | 해시]</code></strong> : 지정한 시점으로 저장소, 스테이지, 작업 디렉터리를 <strong>모두 강제로 되돌립니다.</strong> 이후의 작업물은 사라지므로 매우 신중해야 합니다.</li>
<li><strong><code>git revert [해시]</code></strong> :  기존 이력을 지우지 않고, <strong>특정 커밋의 내용을 반대로 수행하는 새로운 커밋</strong>을 만듭니다.</li>
<li><strong><code>git switch -C [브랜치명]</code></strong> : 새로운 브랜치를 생성함과 동시에 해당 브랜치로 즉시 이동합니다.</li>
<li><strong><code>git merge [브랜치명]</code></strong> : 현재 브랜치에 다른 브랜치의 변경 사항을 합칩니다.</li>
</ul>
