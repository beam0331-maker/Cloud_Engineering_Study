<h2 id="til-git-병합과-github-그리고-java">[TIL] Git 병합과 GitHub, 그리고 Java</h2>
<h3 id="🛠️-part-1-git의-병합">🛠️ Part 1. Git의 병합</h3>
<h4 id="1-fast-forward">1) Fast-forward</h4>
<ul>
<li><strong>이론</strong> : <code>main</code> 브랜치에서 분기한 후, 분기된 브랜치에서 작업을 진행하는 동안 <code>main</code> 브랜치에 추가 커밋이 없을 때.</li>
<li><strong>특징</strong> : <code>main</code>의 HEAD 포인터를 분기된 브랜치의 최신 커밋 지점으로 이동시킵니다.</li>
</ul>
<h4 id="2-병합-커밋">2) 병합 커밋</h4>
<ul>
<li><strong>이론</strong> : main 브랜치에서 변경사항이 없을때 병합하기 위해 최종 커밋이 필요한 상황</li>
<li><strong>특징</strong> : Fast-forward와 다르게 두 브랜치를 합친 '병합 커밋'이 생성된다</li>
</ul>
<h4 id="3-병합-충돌">3) 병합 충돌</h4>
<ul>
<li><strong>이론</strong> : 같은 파일의 같은 위치(같은 )를 서로 다르게 수정하고 병합할 때</li>
<li><strong>특징</strong> : 직접 에디터를 열어 <strong>어떤 내용을 우선할지 결정</strong>하여 수정 후 다시 <code>add</code>와 <code>commit</code>을 해야 완료해야한다.</li>
</ul>
<hr />
<h3 id="🌐-part-2-github--push-fetch-pull">🌐 Part 2. GitHub : Push, Fetch, Pull</h3>
<h4 id="1-push">1) Push</h4>
<ul>
<li><strong>동작</strong>: 로컬 저장소에서 커밋을 GitHub 원격 저장소로 업로드</li>
<li><strong>명령어 흐름</strong>:<ol>
<li><code>$ git remote add origin [원격 저장소]</code> : 원격 저장소 주소를 <code>origin</code>이라는 이름으로 등록합니다.</li>
<li><code>$ git push -u origin main</code> : 로컬의 <code>main</code> 브랜치를 원격에 올립니다.<blockquote>
<p><em>-u</em> : --set-upstream
이 로컬 브랜치는 저 원격 브랜치를 바라보게 하겠다 라는 의미이다.</p>
</blockquote>
</li>
</ol>
</li>
</ul>
<h4 id="2-fetch">2) Fetch</h4>
<ul>
<li><strong>동작</strong>: 원격의 최신 정보를 가져오기만 하며 로컬 디렉토리(실제 파일)에는 변화는 없습니다.</li>
<li><strong>명령어 흐름</strong>:<ol>
<li><code>$ git fetch origin</code> : 원격의 정보를 로컬 저장소로 가져옴.</li>
<li><code>$ git checkout origin/main</code> : 가져온 내용을 체크아웃하여 눈으로 확인.</li>
<li><code>$ git merge origin/main main</code> : 확인 후 안전하다고 판단되면 수동으로 병합.</li>
</ol>
</li>
<li>*<em>merge *</em> : 원하는 브랜치에서 어떤 브랜치를 병합할지 정확히 알아야 한다.</li>
</ul>
<h4 id="3-pull-즉시-반영하기">3) Pull (즉시 반영하기)</h4>
<ul>
<li><strong>동작</strong>: 원격 저장소의 커밋을 가져오면서 현재 브랜치에 병합.</li>
<li><strong>명령어 흐름</strong>:<ol>
<li><code>$ git pull origin main</code> : 원격 내용을 가져와서 즉시적용</li>
</ol>
</li>
</ul>
