<h4 id="🟢-today--자바가-움직이는-방식과-jvm">🟢 Today : 자바가 움직이는 방식과 JVM</h4>
<p>오늘은 자바 개발을 위해 JDK 21을 설치하고, 자바 프로그램이 우리 컴퓨터에서 어떻게 실행되는지 그 과정을 살펴봤습니다.</p>
<ol>
<li>JDK, JRE, JVM</li>
</ol>
<ul>
<li><p>JDK (Java Development Kit) : 자바 개발 종합 도구입니다. 컴파일러와 디버깅 도구가 포함되어 있습니다.</p>
</li>
<li><p>JRE (Java Runtime Environment) : 자바 실행 환경입니다. 컴파일된 프로그램을 실행하기 위한 라이브러리 집합입니다.</p>
</li>
<li><p>JVM (Java Virtual Machine) : 자바 가상 머신입니다. 실제로 프로그램이 돌아가는 가상의 공간입니다.</p>
<p>💡 JDK 21을 선택한 이유
수많은 버전 중 왜 하필 21 버전을 쓰는지 궁금했는데, 이유는 LTS (Long Term Support) 버전이기 때문이었습니다. 기업 환경에서 가장 중요한 <strong>'서비스 안정성'</strong>을 장기적으로 보장받을 수 있는 버전으로 교육이 진행되는 실무적인 디테일이 놀라웠습니다.</p>
</li>
</ul>
<ol start="2">
<li>JVM
자바 강의 중에 가장 흥미로웠던 부분은 바로 <strong>JVM (Java Virtual Machine)</strong>입니다.</li>
</ol>
<ul>
<li><p>OS에 독립적인 실행 : 원래 운영체제마다 알아듣는 말이 다르지만, 자바는 JVM이 중간에서 각 OS (Windows, Linux 등)에 맞춰 실시간으로 통역을 해줍니다.</p>
</li>
<li><p>컴파일과 실행의 흐름 : 우리가 작성한 .java 소스 파일을 javac로 컴파일하면 JVM이 이해할 수 있는 <strong>바이트코드(.class)</strong>가 생성됩니다. 이걸 JVM이 읽어서 우리 컴퓨터의 진짜 CPU가 알아듣게 전달해 주는 방식입니다.</p>
<p>💡 핵심 요약 : JVM은 일종의 가상 CPU이다.
CPU가 기계어를 읽듯이 JVM은 바이트코드를 읽습니다. 특히 이 과정에서 CPU의 물리적인 구조를 소프트웨어로 흉내 냈다는 부분이 흥미로웠습니다.</p>
</li>
</ul>
